# ADR-0009: UI セッション中心モデル

- **Status**: Accepted
- **Date**: 2026-03-23
- **Issue**: [#537](https://github.com/mizunowanko-org/vibe-admiral/issues/537)
- **Tags**: frontend, ui, session, session-chat

## Context

UI の根本原則:

> **チャット欄 = Claude Code セッション1つ とセット**

しかし当時の UI は Dock/Flagship を「特別なタブ」、Ship を「モーダル → 右パネルのカード」と別々に設計しており、以下の問題が繰り返し発生していた:

- Dock/Flagship 間の thinking 状態がリーク
- tool_use 展開時にレイアウトが崩壊
- タブ選択状態が視覚的に不明確
- 入力中チャット・画面状態が勝手にリセット
- フォーカス管理と Ship チャット遷移の不具合

## Decision

全セッション種別（Dock/Flagship/Ship/Dispatch）を同一の `SessionChat` コンポーネントで扱うセッション中心モデルに再設計する。

### セッション種別と表示ルール

| セッション種別 | 入力欄 | カード表示 | 備考 |
|--------------|--------|-----------|------|
| Dock | あり | 右パネルにカード | Commander（対話型） |
| Flagship | あり | 右パネルにカード | Commander（対話型） |
| Ship | なし | 右パネルにカード | 非対話型（`-p` モード） |
| Dispatch | なし | 親セッション内にサブカード | サブエージェント |

### UI レイアウト

- **右パネル**: 全セッションがカードとして並ぶ。フォーカスは全カードで排他的に1つ
- **左パネル**: フォーカスされたセッションのチャットを表示
- カードクリックで即座に切り替え（「戻る」ボタン不要）
- Dock/Flagship のカードは常に上部に固定表示
- 各セッションは独立した入力状態（下書き）を持つ

### 実装方針

- `Bridge` コンポーネントを廃止し、汎用の `SessionChat` コンポーネントに置換
- `SessionChat` は `sessionId` を受け取り、セッションのチャット履歴と入力欄を管理
- 入力欄の有無は `sessionType` prop で制御
- Zustand store で `focusedSessionId` を管理し、排他的フォーカスを実現

### 検討したが採用しなかった案

| 案 | 不採用理由 |
|----|----------|
| セッション種別ごとに専用コンポーネント | コンポーネント共有の欠如が既存バグの根本原因。同じ問題を再発させる |
| モーダル表示の継続 | 複数 Ship の並行閲覧ができない。カードベースの方が一覧性が高い |

## Consequences

- **Positive**: 全セッション種別が同一コンポーネントで描画され、表示ロジックの一貫性が保証される
- **Positive**: セッション間の状態リークが構造的に排除される
- **Positive**: フォーカス管理が `focusedSessionId` の一元管理に集約され、不整合が起きない
- **Negative**: 既存の Bridge/Ship 専用 UI の全面書き直しが必要
- **Negative**: セッション種別ごとの表示差異を `SessionChat` 内の条件分岐で処理するため、条件が増えると複雑化する可能性がある
